# 1. 项目概述

## 1.1 软件定位

本软件定位为轻量级飞行器 CSV 数据可视化桌面程序，用于读取一个或多个 CSV 文件，并对飞行器运行过程中的三维轨迹、三轴速度、三轴姿态角、三轴角速度以及 CSV 中记录的其他状态量进行可视化和对比。

软件第一版应优先满足“打开即用、配置清晰、多文件对比方便、打包后可双击运行”的目标，不追求大型后处理平台的复杂分析功能。

## 1.2 目标用户与使用场景

- 飞行器仿真、控制、热力学或气动相关实验结果的快速查看。

- 同一算法在不同参数设置下的飞行状态对比。

- 多个 CSV 实验结果之间的速度、姿态、角速度和轨迹对比。

- 对 CSV 中其他状态量进行横向或纵向对比，例如温度、压力、体积、浮力、环境密度等。

## 1.3 MVP 范围

| **类别**   | **第一版实现**                                                   | **暂不实现/后续考虑**                |
|------------|------------------------------------------------------------------|--------------------------------------|
| 数据读取   | 读取一个或多个 CSV；加载时可设置别名；自动列识别；必要时列映射   | 实时数据流、数据库管理               |
| 三维轨迹   | 左侧固定 3D 轨迹图；多文件轨迹对比；起点/终点；最多显示 50000 点 | 地图背景、坐标系转换、飞行器实体模型 |
| 默认图窗   | 右侧默认 3×3，固定显示速度、姿态、角速度                         | 默认图窗中自定义添加曲线             |
| 自定义图窗 | 新增选项卡、设置行列、横向对比、纵向对比、手动添加曲线           | 复杂滤波、单位转换、动画播放         |
| 配置保存   | 工程 JSON 保存/加载；重要操作后自动保存                          | 多人协作、云同步                     |
| 导出       | 单个二维图窗导出 PNG；三维轨迹导出 PNG                           | 整页导出、批量导出、报告生成         |

# 2. 数据格式与约束

## 2.1 样例 CSV 结构

当前样例 CSV 文件包含 5001 行、20 列，列名如下：

time  
x, y, z  
vx, vy, vz  
phi, theta, psi  
wx, wy, wz  
T_f, T_in, volume, buoyancy, pressure, rho_a, T_a

| **类别**   | **示例列**                                        | **默认用途**                                |
|------------|---------------------------------------------------|---------------------------------------------|
| 时间       | time                                              | 二维图窗默认横坐标                          |
| 位置       | x, y, z                                           | 三维轨迹绘制                                |
| 速度       | vx, vy, vz                                        | 默认 3×3 第一行图窗                         |
| 姿态角     | phi, theta, psi                                   | 默认 3×3 第二行图窗，第一版默认单位为 rad   |
| 角速度     | wx, wy, wz                                        | 默认 3×3 第三行图窗，第一版默认单位为 rad/s |
| 其他状态量 | T_f, T_in, volume, buoyancy, pressure, rho_a, T_a | 通过自定义图窗绘制                          |

## 2.2 数据规模限制

| **项目**         | **默认限制** | **处理策略**                                         |
|------------------|--------------|------------------------------------------------------|
| 单个 CSV 行数    | 100000 行    | 超过限制时提示用户；第一版可拒绝加载或要求确认后加载 |
| 同时加载文件数   | 10 个        | 超过限制时提示用户；后续可在设置中开放修改           |
| 三维轨迹显示点数 | 50000 点     | 超过时自动抽稀显示；原始数据仍完整保留               |
| 自定义选项卡布局 | 最多 5×5     | 避免界面过度拥挤和绘图性能下降                       |

## 2.3 默认单位

- 姿态角默认按弧度制解释，标签可显示为 phi / rad、theta / rad、psi / rad。

- 角速度默认按 rad/s 解释，标签可显示为 wx / rad/s、wy / rad/s、wz / rad/s。

- 第一版不做自动单位转换，后续可扩展 deg/rad 与 deg/s/rad/s 的转换。

# 3. 主界面布局设计

## 3.1 总体布局

主界面采用“左侧三维轨迹 + 右侧选项卡二维图窗 + 底部公共图例”的布局。左侧宽度约占整体 2/5，右侧选项卡区域约占 3/5。

┌─────────────────────────────────────────────────────────────┐  
│ 菜单栏 / 工具栏 │  
├───────────────────────┬─────────────────────────────────────┤  
│ │ \[基本状态量\] \[+\] │  
│ │ ┌────────┬────────┬────────┐ │  
│ 三维轨迹 │ │ vx │ vy │ vz │ │  
│ │ ├────────┼────────┼────────┤ │  
│ │ │ phi │ theta │ psi │ │  
│ │ ├────────┼────────┼────────┤ │  
│ │ │ wx │ wy │ wz │ │  
│ │ └────────┴────────┴────────┘ │  
├───────────────────────┴─────────────────────────────────────┤  
│ 公共图例区：● data1 ● data2 ● data3 │  
└─────────────────────────────────────────────────────────────┘

## 3.2 菜单栏与工具栏

| **菜单/按钮** | **功能**                                        |
|---------------|-------------------------------------------------|
| 打开 CSV      | 选择一个或多个 CSV 文件，进入导入与别名设置流程 |
| 数据文件管理  | 查看已加载文件、修改别名/颜色/列映射、删除文件  |
| 保存工程      | 将当前配置保存为 JSON 工程文件                  |
| 工程另存为    | 选择新的 JSON 路径保存工程                      |
| 加载工程      | 从 JSON 恢复文件、图窗、图例、列映射和配置      |
| 导出图像      | 导出当前二维图窗或三维轨迹为 PNG                |
| 设置          | 预留全局设置，例如最大文件数、抽稀点数、主题等  |

## 3.3 底部公共图例区

底部区域不显示数据量、鼠标坐标等状态信息，而用于展示公共图例。每个已加载 CSV 文件对应一个图例项，包含颜色和别名。

| **操作**   | **行为**                                            |
|------------|-----------------------------------------------------|
| 单击图例项 | 切换该文件的显示/隐藏状态，并触发最后操作优先机制   |
| 双击别名   | 重命名该 CSV 文件别名                               |
| 点击颜色块 | 修改该 CSV 的全局默认颜色                           |
| 右键菜单   | 重命名、修改颜色、显示/隐藏、重新列映射、从工程移除 |

**设计原则：**文件别名优先显示，完整文件路径只在文件管理窗口中显示。

# 4. CSV 导入、别名与列映射

## 4.1 导入流程

用户点击“打开 CSV”  
↓  
选择一个或多个 CSV 文件  
↓  
弹出“导入 CSV 数据文件”窗口  
↓  
系统默认分配别名 data1、data2、data3……  
↓  
用户可修改别名  
↓  
执行列名自动识别  
↓  
识别失败或部分缺失时弹出列映射窗口  
↓  
加入工程，刷新三维轨迹、默认图窗和公共图例

## 4.2 CSV 别名规则

- 加载文件时默认命名为 data1、data2、data3 等。

- 别名在当前工程内必须唯一，不能为空。

- 用户可在导入窗口、公共图例或文件管理窗口中修改别名。

- 别名编号只递增，不因删除文件而复用。例如删除 data2 后再加载新文件，默认命名为 data4。

## 4.3 列名自动识别规则

| **标准字段**     | **优先匹配列名**                                                           |
|------------------|----------------------------------------------------------------------------|
| time             | time, t, Time, timestamp                                                   |
| x, y, z          | x/y/z, pos_x/pos_y/pos_z, position_x/position_y/position_z                 |
| vx, vy, vz       | vx/vy/vz, v_x/v_y/v_z, vel_x/vel_y/vel_z, velocity_x/velocity_y/velocity_z |
| roll, pitch, yaw | phi/theta/psi, roll/pitch/yaw, att_x/att_y/att_z                           |
| wx, wy, wz       | wx/wy/wz, p/q/r, omega_x/omega_y/omega_z, ang_vel_x/ang_vel_y/ang_vel_z    |

## 4.4 列映射窗口

当软件无法识别典型列名，或某些标准字段缺失时，应弹出列映射窗口，允许用户手动选择每个标准字段对应的实际 CSV 列。每个映射项允许选择“无”，以支持缺少某类数据的 CSV。

# 5. 三维轨迹图窗设计

左侧三维轨迹图窗是固定区域，用于绘制所有已加载 CSV 文件的三维运动轨迹。第一版默认使用 x、y、z 三列，不进行经纬度或坐标系转换。

| **功能**     | **第一版规则**                                  |
|--------------|-------------------------------------------------|
| 轨迹数据来源 | 各文件 column_mapping 中的 x、y、z 字段         |
| 坐标轴标签   | 默认 X、Y、Z，后续可在设置中修改                |
| 多文件对比   | 每个 CSV 文件绘制一条轨迹，颜色使用文件全局颜色 |
| 起点/终点    | 默认显示起点和终点标记                          |
| 网格         | 默认开启                                        |
| 等比例坐标   | 建议实现 equal_axis 选项                        |
| 最大显示点数 | 50000 点，超过时自动抽稀                        |
| 公共图例联动 | 遵循最后操作优先机制                            |

# 6. 二维图窗系统设计

## 6.1 图窗类型

软件中的二维图窗分为预设图窗和自定义图窗两类。二者在功能权限上必须明确区分。

| **图窗类型**          | **来源**                            | **配置能力**                                          |
|-----------------------|-------------------------------------|-------------------------------------------------------|
| 预设图窗 PresetPlot   | 默认“基本状态量”选项卡中的 9 个图窗 | 固定绘制对应状态量，只允许修改绘图细节和局部显示/隐藏 |
| 自定义图窗 CustomPlot | 用户新建选项卡中的图窗              | 支持横向对比、纵向对比、手动添加曲线、完整曲线管理    |

## 6.2 图窗区域按钮

每个图窗区域右下角固定提供两个小按钮：

\[曲线\] \[设置\]

| **按钮** | **预设图窗行为**                                              | **自定义图窗行为**                                      |
|----------|---------------------------------------------------------------|---------------------------------------------------------|
| 曲线     | 打开简化曲线显示管理窗口，仅允许显示/隐藏当前图窗中各文件曲线 | 打开完整曲线管理窗口，可显示/隐藏、删除、排序、修改样式 |
| 设置     | 打开绘图细节设置窗口，不提供绘图模式和添加曲线功能            | 打开多选项卡配置窗口，管理绘图模式、绘图细节、添加曲线  |

## 6.3 默认 3×3 基本状态量图窗

| **位置**       | **图窗标题** | **固定标准变量** | **默认 Y 轴标签** |
|----------------|--------------|------------------|-------------------|
| 第 1 行第 1 列 | X 方向速度   | vx               | vx                |
| 第 1 行第 2 列 | Y 方向速度   | vy               | vy                |
| 第 1 行第 3 列 | Z 方向速度   | vz               | vz                |
| 第 2 行第 1 列 | 滚转角       | roll             | phi / rad         |
| 第 2 行第 2 列 | 俯仰角       | pitch            | theta / rad       |
| 第 2 行第 3 列 | 偏航角       | yaw              | psi / rad         |
| 第 3 行第 1 列 | X 轴角速度   | wx               | wx / rad/s        |
| 第 3 行第 2 列 | Y 轴角速度   | wy               | wy / rad/s        |
| 第 3 行第 3 列 | Z 轴角速度   | wz               | wz / rad/s        |

- 预设图窗固定绘制所有已加载 CSV 文件中的对应状态量。

- 预设图窗不支持添加曲线、删除曲线、修改变量或切换横向/纵向对比。

- 新增 CSV 后，如果有对应列或映射，则自动加入预设图窗。

- 某文件缺少对应变量时跳过，不应导致整个图窗报错。

## 6.4 预设图窗设置窗口

预设图窗的“设置”窗口只提供绘图细节，不显示绘图模式、横向/纵向对比或添加曲线。

| **设置项**    | **是否支持**                     |
|---------------|----------------------------------|
| 图窗标题      | 支持                             |
| X/Y 轴标签    | 支持                             |
| 网格开关      | 支持                             |
| 局部图例开关  | 支持，默认关闭                   |
| X/Y 轴范围    | 支持自动或手动                   |
| 默认线宽/线型 | 支持，对该预设图窗内所有曲线生效 |
| 修改绘制变量  | 不支持                           |
| 添加/删除曲线 | 不支持                           |

## 6.5 预设图窗曲线窗口

预设图窗的“曲线”窗口仅用于控制当前图窗中各文件曲线的显示/隐藏。

曲线显示管理：X方向速度  
  
显示 文件别名 实际X列 实际Y列  
\[✓\] data1 time vx  
\[✓\] data2 time vx  
\[ \] data3 time vx  
  
\[全选\] \[全不选\] \[恢复默认\] \[关闭\]

# 7. 自定义选项卡与自定义图窗

## 7.1 新建选项卡

用户点击选项卡栏旁的“+”按钮，弹出新建选项卡窗口。用户输入名称、行数、列数后生成自定义图窗网格。

名称：\[ 热力学状态 \]  
行数：\[ 2 ▼ \]  
列数：\[ 3 ▼ \]  
  
约束：1 \<= 行数 \<= 5，1 \<= 列数 \<= 5

- 自定义选项卡中的图窗初始允许为空白。

- 空白图窗显示“未配置图窗”，用户可后续点击“设置”配置。

## 7.2 自定义图窗配置窗口

自定义图窗的“设置”窗口采用多选项卡结构，避免单一窗口过于臃肿。第一版建议实现三个配置页：绘图模式、绘图细节、添加曲线。

| **配置页** | **功能**                                                 |
|------------|----------------------------------------------------------|
| 绘图模式   | 管理横向对比、纵向对比或手动模式                         |
| 绘图细节   | 管理标题、坐标轴、网格、局部图例、坐标范围、默认线型线宽 |
| 添加曲线   | 单独指定某文件中的某一列加入当前图窗                     |

## 7.3 横向对比模式

横向对比用于“多个 CSV 文件对比同一个变量”。用户选择一个变量或列名，软件在所有参与文件中查找并绘制该变量。

| **设置项**           | **说明**                               |
|----------------------|----------------------------------------|
| 对比方式             | 按标准变量映射 / 按实际列名            |
| X 轴                 | 使用每个文件的时间映射，或指定实际列名 |
| Y 轴变量/列名        | 如 vx、T_f、pressure 等                |
| 参与文件             | 可勾选参与横向对比的 CSV               |
| 自动包含后续加载 CSV | 默认可开启；新文件包含该列时自动加入   |
| 缺失列处理           | 跳过缺失文件并提示，不中断绘图         |

**默认图例策略：**横向对比默认关闭局部图例，依赖底部公共图例区解释文件颜色。

## 7.4 纵向对比模式

纵向对比用于“同一个 CSV 文件内部对比多个变量”。用户选择一个文件，再选择该文件中的多个 Y 轴列。

| **设置项** | **说明**                            |
|------------|-------------------------------------|
| 数据文件   | 选择一个 CSV 文件，例如 data1       |
| X 轴列     | 默认 time，也可选择该文件中的其他列 |
| Y 轴列     | 多选，例如 T_f、T_in、T_a           |
| 颜色策略   | 默认每个变量自动分配颜色            |
| 局部图例   | 默认开启，用于区分同文件内多个变量  |

纵向对比模式下切换文件时，软件应尝试复用原先选择的 Y 列名：

| **情况**             | **处理方式**                                         |
|----------------------|------------------------------------------------------|
| 新文件全部包含原列   | 自动切换并绘制这些列                                 |
| 新文件部分包含原列   | 绘制可用列，提示缺失列，等待用户选择替代列或确认跳过 |
| 新文件全部不包含原列 | 不自动绘制，提示用户重新选择 Y 轴列                  |
| 新文件缺少 X 轴列    | 提示用户重新选择 X 轴列                              |

## 7.5 添加单独曲线

自定义图窗应提供“添加曲线”页，用于单独指定某 CSV 文件中的某一列并加入当前图窗。该功能用于处理横向/纵向对比无法覆盖的不规则需求。

数据文件：\[ data1 ▼ \]  
X轴列：\[ time ▼ \]  
Y轴列：\[ pressure ▼ \]  
曲线名称：\[ data1 / pressure \]  
颜色模式：(●) 使用文件默认颜色 ( ) 自定义颜色  
线宽：\[ 1.5 \]  
线型：\[ 实线 ▼ \]  
  
\[添加到当前图窗\]

## 7.6 自定义图窗曲线管理窗口

自定义图窗点击“曲线”按钮后，打开完整曲线管理窗口。该窗口展示当前图窗所有曲线，包括横向生成、纵向生成和手动添加的曲线。

| **功能**      | **说明**                                                    |
|---------------|-------------------------------------------------------------|
| 显示/隐藏     | 控制单条曲线可见性，并参与最后操作优先机制                  |
| 删除曲线      | 删除选中曲线                                                |
| 排序          | 调整绘制顺序                                                |
| 修改颜色模式  | 使用文件颜色或自定义颜色                                    |
| 修改线宽/线型 | 用于自定义曲线或批量样式调整                                |
| 查看来源      | 标明曲线来自 horizontal_compare、vertical_compare 或 manual |

# 8. 可见性优先级机制

软件中存在两类显示/隐藏操作：底部公共图例的文件级显示/隐藏，以及图窗“当前曲线”窗口中的曲线级显示/隐藏。设计要求采用“最后一次操作优先”机制，而不是简单的全局与局部逻辑相与。

## 8.1 行为示例

| **操作顺序**                                              | **最终结果**                                          |
|-----------------------------------------------------------|-------------------------------------------------------|
| 先在公共图例隐藏 data1，再在 vx 图窗曲线窗口显示 data1/vx | data1/vx 显示；data1 在其他图窗和三维轨迹中仍保持隐藏 |
| 先在 vx 图窗隐藏 data1/vx，再在公共图例显示 data1         | data1 的所有关联曲线和三维轨迹显示，包括 data1/vx     |

## 8.2 推荐实现方式

为实现最后操作优先，文件级可见性和曲线级可见性都应保存最后修改时间。绘图时比较时间戳，使用较晚一次操作的 visible 值。

def resolve_curve_visible(file_config, curve_visibility):  
if curve_visibility.modified_at \> file_config.visibility_modified_at:  
return curve_visibility.visible  
return file_config.visible

对于预设图窗，曲线局部可见性可保存为每个 plot 对每个 file_id 的局部状态：

"curve_visibility": {  
"file_001": {  
"visible": true,  
"modified_at": "2026-07-01T16:21:30"  
}  
}

# 12. 自动保存、错误处理与导出

## 12.1 自动保存策略

- 重要操作后标记 project_dirty = true。

- 采用约 1 秒防抖写入，连续修改时只在最后写一次 JSON。

- 自动保存失败时提示用户，并保持 dirty 状态。

- 同时提供“保存工程”和“工程另存为”手动操作。

| **操作**                 | **是否自动保存** |
|--------------------------|------------------|
| 添加/删除 CSV            | 是               |
| 修改别名、颜色、列映射   | 是               |
| 公共图例显示/隐藏        | 是               |
| 局部曲线显示/隐藏        | 是               |
| 添加/删除/重命名选项卡   | 是               |
| 修改图窗绘图细节         | 是               |
| 添加/删除自定义曲线      | 是               |
| 鼠标缩放、平移、旋转视角 | 否               |

## 12.2 路径失效处理

加载工程时，如果某个 CSV 文件路径不存在，软件应提示文件缺失，允许用户重新定位该文件。如果用户选择跳过，则保留该文件配置但不绘制；重新定位成功后恢复其别名、颜色、列映射和相关曲线配置。

## 12.3 导出功能

第一版导出范围为单个二维图窗 PNG 和三维轨迹 PNG。选项卡整体导出、主界面截图、批量导出等功能放入第二阶段。
